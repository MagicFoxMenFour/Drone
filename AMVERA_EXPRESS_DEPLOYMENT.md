# Развертывание на Amvera с Express.js и SQLite

## 📋 Важные отличия от Vercel

**Это НЕ Serverless Functions!** На Amvera:
- ✅ Полноценное Node.js приложение с Express.js сервером
- ✅ SQLite база данных в постоянном хранилище (`/data/`)
- ✅ Автоматическое восстановление при перезапуске
- ✅ Простое масштабирование ресурсов

**GitHub интеграция удалена** - все данные теперь в SQLite базе данных на сервере.

## 🚀 Быстрый старт

### Шаг 1: Подготовка репозитория

```bash
# Убедитесь что репозиторий на GitHub
git remote -v  # должен показать origin

# Пушьте основную ветку
git branch -M main
git push -u origin main
```

### Шаг 2: Создание проекта на Amvera

1. Перейдите на https://console.amvera.io
2. Нажмите **"Создать приложение"** → **"GitHub"**
3. Авторизуйтесь с GitHub
4. Выберите ваш репозиторий
5. Выберите ветку **main**
6. Нажмите **"Далее"**

### Шаг 3: Конфигурация

На шаге конфигурации Amvera должен автоматически найти `amvera.yaml`. Если нет:

1. **Build command**: `npm run build`
2. **Run command**: `npm start`
3. **Container port**: `3000`

### Шаг 4: Переменные окружения

Добавьте переменные в Amvera Console → **Environment Variables**:

```
ADMIN_LOGIN=admin
ADMIN_PASSWORD=your_secure_password_12chars_or_more
NODE_ENV=production
PORT=3000
DB_PATH=/data/app.db
RESEND_API_KEY=(опционально)
RESEND_FROM=(опционально)
LEADS_TO_EMAIL=(опционально)
```

### Шаг 5: Развертывание

1. Нажмите **"Развернуть"**
2. Дождитесь завершения сборки (обычно 2-5 минут)
3. Перейдите на ваш домен
4. Откройте `/admin/login`

## 💾 База данных SQLite

### Автоматическое создание таблиц

При первом запуске приложение автоматически создает все необходимые таблицы:

- `services` - услуги
- `cases` - проекты/кейсы
- `blog_posts` - статьи блога
- `about_page` - информация о компании
- `employees` - сотрудники команды
- `leads` - заявки с сайта
- `admin_users` - пользователи админ панели

### Расположение БД

На Amvera БД находится в постоянном хранилище:
```
/data/app.db
```

Это означает что данные сохраняются даже при перезапуске приложения!

### Резервное копирование

Amvera автоматически создает бэкапы `/data` папки. Смотри Backups в Amvera Console.

## 📍 API Endpoints

### Публичные эндпоинты

```
GET /api/content              # Получить все публичные данные
```

Возвращает:
```json
{
  "services": [...],          // Только с published=1
  "cases": [...],             // Только с published=1
  "blog_posts": [...],        // Только с published=1
  "about_page": {...},        // Единственная запись
  "employees": [...]          // Только с active=1, сортированы по sort
}
```

### Заявки

```
POST /api/lead               # Создать новую заявку
```

Параметры:
```json
{
  "name": "Иван",
  "email": "ivan@example.com",
  "phone": "+79991234567",
  "service": "Аэросъемка",
  "message": "Хочу снять дом"
}
```

### Админ API (требуется аутентификация)

```
POST /api/admin/login              # Вход
POST /api/admin/logout             # Выход
GET /api/admin/me                  # Текущий пользователь
GET /api/admin/dashboard           # Статистика

GET /api/admin/services            # Получить все услуги
POST /api/admin/services           # Создать услугу
GET /api/admin/services/:id        # Получить услугу
PATCH /api/admin/services/:id      # Обновить услугу
DELETE /api/admin/services/:id     # Удалить услугу

# Аналогично для:
# - /api/admin/cases
# - /api/admin/blog_posts
# - /api/admin/about_page
# - /api/admin/employees
# - /api/admin/leads
```

## 🚨 Решение проблем

### "502/503 Bad Gateway"

**Причины:**
- Приложение еще стартует (смотрите логи)
- Ошибка в коде приложения
- Недостаточно памяти/ресурсов

**Решение:**
1. Откройте **Logs** в Amvera Console
2. Найдите ошибку
3. Исправьте и пушьте изменения

### "Cannot connect to database"

**Причины:**
- Папка `/data` еще не создана
- Проблемы с правами доступа

**Решение:**
- Amvera автоматически создает `/data` при первом запуске
- Дождитесь полного запуска приложения

### "Неверный логин или пароль"

**Причины:**
- Неправильно установлены `ADMIN_LOGIN` и `ADMIN_PASSWORD`
- Приложение еще стартует

**Решение:**
1. Проверьте переменные в Environment Variables
2. Очистите куки браузера
3. Попробуйте еще раз

### "Заявки не приходят на email"

**Причины:**
- Не установлены `RESEND_API_KEY` и `LEADS_TO_EMAIL`
- Домен не верифицирован на Resend
- Неправильный `RESEND_FROM`

**Решение:**
1. Проверьте все три переменные
2. Убедитесь что домен верифицирован на https://resend.com
3. Используйте формат `Domain <noreply@domain.com>`
