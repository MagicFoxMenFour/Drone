# Админ панель - Краткое руководство

## 📋 Что реализовано

Ваш проект **полностью готов** к развертыванию! Все требуемые функции реализованы:

### ✅ Авторизация
- Защита админ панели через логин и пароль
- Безопасные HTTP-only cookies
- Автоматическое истечение сессии через 7 дней
- Защита от CSRF атак

### ✅ Управление контентом
- **Услуги** - добавление, редактирование, удаление
- **Кейсы/Портфолио** - полный CRUD
- **Блог** - управление статьями
- **О нас** - редактирование информации о компании
- **Сотрудники** - добавление/удаление членов команды
- **Заявки** - просмотр, изменение статуса, удаление

### ✅ Email интеграция
- Автоматическая отправка заявок на почту через Resend
- Дублирование всех данных формы
- Красивое форматирование писем

### ✅ Хранилище данных
- GitHub как хранилище (JSON файлы в ветке `content`)
- Все изменения автоматически коммитятся
- История всех изменений в Git

## 🚀 Быстрый старт

### Локально

```bash
# 1. Установите зависимости
npm install

# 2. Настройте .env (скопируйте из .env.example)
cp .env.example .env
# Отредактируйте .env с вашими данными

# 3. Запустите с API функциями
npm install -g vercel
vercel dev

# 4. Откройте в браузере
# http://localhost:3000/admin/login
```

### На Amvera

1. Создайте аккаунт на https://console.amvera.io
2. Следуйте [инструкции AMVERA_DEPLOYMENT.md](./AMVERA_DEPLOYMENT.md)
3. Займет ~5 минут!

## 📍 Адреса админ панели

### Локально
- Вход: http://localhost:3000/admin/login
- Панель: http://localhost:3000/admin

### На Amvera
- Вход: https://yourdomain.amvera.io/admin/login
- Панель: https://yourdomain.amvera.io/admin

## 🔑 Переменные окружения

Обязательные:
```env
ADMIN_LOGIN=admin
ADMIN_PASSWORD=your_password
ADMIN_SESSION_SECRET=random_32_chars

GITHUB_OWNER=your_username
GITHUB_REPO=repo_name
GITHUB_TOKEN=ghp_xxxxx
GITHUB_CONTENT_BRANCH=content
```

Опционально (для email):
```env
RESEND_API_KEY=re_xxxxx
RESEND_FROM=noreply@yourdomain.com
LEADS_TO_EMAIL=admin@yourdomain.com
```

## 💾 Где хранятся данные

Все данные хранятся в GitHub ветке `content`:

```
content/
├── services.json       # Услуги
├── cases.json          # Кейсы
├── blog_posts.json     # Статьи
├── about_page.json     # Информация о компании
├── employees.json      # Сотрудники
└── leads.json          # Заявки
```

Каждое изменение через админ панель автоматически коммитится в GitHub!

## 🔄 Как работает

1. **Вход** → Проверка логина/пароля
2. **Создание сессии** → Сохранение в httpOnly cookie
3. **Запрос к контенту** → Чтение из GitHub
4. **Редактирование** → Отправка на API
5. **Сохранение** → Запись в GitHub + коммит
6. **Email** → Отправка заявки на почту (опционально)

## 🐛 Частые ошибки

### "Unauthorized" при входе
- Проверьте логин/пароль в `.env`
- Очистите куки браузера

### "GitHub token invalid"
- Создайте новый PAT токен на GitHub
- Убедитесь что он не истек
- Обновите в `.env`

### "Cannot save data to GitHub"
- Проверьте что ветка `content` существует
- Убедитесь что у токена права на `repo`

### API не отвечает
- Убедитесь что запущен `vercel dev`, а не `npm run dev`
- Проверьте консоль (F12) на ошибки

## 📊 Примеры использования

### Добавление новой услуги

1. Откройте `/admin/services`
2. Нажмите "Добавить"
3. Заполните форму
4. Нажмите сохранить
5. Данные автоматически появятся на публичном сайте

### Добавление сотрудника

1. Откройте `/admin/employees`
2. Нажмите "Добавить"
3. Заполните имя, роль, био
4. Выберите цвет аватара
5. Сохраните - готово!

### Просмотр заявок

1. Откройте `/admin/leads`
2. Видите всех, кто оставил заявку с сайта
3. Можете изменить статус (new → in_progress → done)
4. Или удалить заявку
